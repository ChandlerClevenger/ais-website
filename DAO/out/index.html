dao_test.js

Test file for the unit and integration tests of the DAO. By calling the main test function, this file automatically deletes all dynamic data from the tables so that we can properly test the functions. Unit tests: These tests implement the stub mode. Which when true, these tests don't actually implement the full functions. They don't connect to the database. These unit tests test two things: if they catch false input and if they can be called without connecting to the mysql database. Integration tests: These tests implement the full function (connection to the database and everything). These test whether the connection passes and if the return value is correct.
